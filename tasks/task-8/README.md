# GitHub CI

Напишите базовый CI/CD для репозитория.

Для этого форкните его на github, создайте `.github/workflows/test.yml` файл и в нем опишите workflow.

Пайплайн должен:

- Запускаться на push и создание PR, не запускаться если закомичены изменения в .md файлах
- Состоять из четырех шагов выполняющихся последовательно: install, lint, build, test
- Запускаться на ubuntu-latest
- Тесты должны запуститься на 22 и 24 ноде
- Переиспользовать кэш пакетного менеджера

Для проверки создайте Pull Request в репозиторий и призывайте @kholstinin на ревью.
